About HBnB
✖
Part1: This project entailed developing an AirBnB clone, HBnB. The project involved all the four main basic requirements of any project, ie, CRUD, create, read, update, delete. The first part involved developing the console of the project, ie, create a terminal for the project using python's CMD module. Well, this is helpful to help you manage the projet easily in different environments. This part entailed developing the BaseClass model. This class defines all the common attributes/methods for other classes. The class entailed public instance attributes, ie id which was made using UUID module, and stringified. The goal was having a unique ID for each BaseModel, eg, a new user is created, they're going to have a unique id that maps all the other attributes if the user to the user. This is used mostly for easier database manipulation and storage. The class also contained updated_at and created_at public instance attributes which assign the current datetime when an instance is created and updated. The class also contained the pulic instance method save that updates the instance attribute updated_at with the current datetime. The project seeked to store classes in form of dictionaries, well the dictionary filestorage system was what was implemented first. The tests for this project were witten using python's unittest module.
Part 2: The second part of the project was designing and developing a database system, the database used for this project was the sql based database system. I created a new user for the project and database for the project. Then added two features to the console to enable saving and deleting of objects stored in a class. The models were all linked together using the unique id in the database.
Part 3: The third part involved developing the front end part of the AirBnB project in static form. No data was served from the database for this part.
Part 4: The fourth part involved the integration of a framework into the AirBnB project. The web framework used for this project was Flask. This is used to help serve create routes to help serve the data.
Part 5: The fifth part of the project involved designing and creating API endpoints, it to get and serve the data from the database storage system. The API used for this case is the REST API.
Part 6: The sixth part of the project was serving data to the front end of the project in and making the whole project dynamic.
- The languages and technologies used for this project were:
* Python, to structure and build the backend of the web. OOP was a large part of it and modules such as:
  • Flask webframework.
  • Python's UUID module.
  • Python's CMD module.
  • Python's unittest module.
* HTML5, CSS & JavaScript to build, style, and make the front end dynamic.
* MySQL to structure and build the web application database.
* The web application was deployed in a linux server running Nginx and Gunicorn.
* The data was served in JSON format
